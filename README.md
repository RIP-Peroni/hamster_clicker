# Инструкция по локальному запуску Hamster Clicker
## Требования для запуска
1. Docker (версия 20.10 или выше)
2. Docker Compose (версия 2.0 или выше)
3. Git
4. Порты: 8080, 5432, 3000 должны быть свободны

## Команды для сборки и запуска проекта
1. Клонирование репозитория

```git clone git@github.com:RIP-Peroni/hamster_clicker.git```

```cd hamster_clicker```


2. Настройка окружения

```cp .env.example .env```

```docker-compose up -d --build```

3. Выполнение миграций

Миграции выполняются автоматически при запуске, но если нужно выполнить вручную:

```docker-compose exec app php artisan migrate```

4. Доступы и тестовый пользователь

Тестового пользователя можно создать используя сидер:

```docker-compose exec app php artisan db:seed```

Данные тестового пользователя:

Email: test@example.com

Пароль: password

Создание нового пользователя через интерфейс:
* Откройте http://localhost:8080/register
* Заполните форму регистрации
* Нажмите "Register"

Как посмотреть на основной функционал
1. Вход в систему
* Откройте http://localhost:8080/login
* Введите email: test@example.com
* Введите пароль: password
* Нажмите "Log in"

2. Клики по хомяку

На странице Dashboard есть изображение хомяка

Кликайте по нему - счётчики статистики будут обновляться

Статистика показывает:
* Общее количество кликов
* Количество кликов за сегодня
* Количество кликов за последние 7 дней с разбивкой по дням

3. Скачивание PDF отчёта
* Нажмите кнопку "Download PDF Report"
* Браузер скачает файл hamster-report.pdf
* Откройте PDF - в нём будет статистика пользователя

4. Выход из системы

Нажмите "Logout" в левом нижнем углу


## Технические решения
### Backend
* Laravel 13 - современный PHP-фреймворк
* PostgreSQL 18.4 - используется как основная БД
* Gotenberg - отдельный сервис для генерации PDF через Chromium

### Frontend
* Vue 3.5 (Composition API) - современный реактивный фреймворк
* Inertia.js - мост между Laravel и Vue, позволяющий создавать SPA без отдельного API
* Tailwind CSS - утилитарный CSS-фреймворк для быстрой стилизации

### Ключевые архитектурные решения
* Использование Inertia.js вместо классического SPA
* Отказ от Sanctum в пользу сессионной аутентификации
* Выделение бизнес-логики в отдельный сервис - StatsService для расчета статистики
* Генерация PDF через Gotenberg
* Обновление статистики без перезагрузки страницы
* Docker-конфигурация без Nginx в разработке
* Составной индекс (user_id, clicked_at) для быстрых агрегаций по датам

### Что можно улучшить в будущем

2. WebSocket для real-time обновлений
3. График активности вместо таблицы
